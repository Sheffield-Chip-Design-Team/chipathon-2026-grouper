# AHB SPI Master

**Owner:** Safaa
**Status:** RTL design starting (per [Schematic Review](../../Schematic%20Review.md) §4, one of the three peripherals — SPI M, SPI S, QSPI — where RTL design has begun). No RTL committed yet under `hw/rtl/`.

**Related:** [Grouper SoC Specification](../Grouper%20SoC%20Specification.md) — memory map, boot sequence (SPI M is a candidate post-boot storage path) | [SPI Master Verification Plan](../../verification/blocks/SPI%20Master%20Verification%20Plan.md)

## Purpose

SPI master peripheral that lets the PicoRV32 CPU issue SPI transactions to an external SPI/QPI memory device (APS6404L-compatible command set), via an AHB-Lite slave interface. Fits the boot-sequence note that SPI (and QSPI) may be used for extended external storage once the initial UART-loaded program is running (see [Grouper SoC Specification § Boot Sequence](../Grouper%20SoC%20Specification.md#boot-sequence)).

## Protocols / Standards Conformity

| ID | Requirement |
|---|---|
| `GRPR-SPIM-001` | AHB-Lite subordinate interface on the CPU side. |
| `GRPR-SPIM-002` | SPI mode 0 (and mode 3, see `GRPR-SPIM-006`), MSB-first, on the external device side. |
| `GRPR-SPIM-003` | Command/transaction set shall be compatible with the APS6404L datasheet's SPI-mode command encoding. |
| `GRPR-SPIM-004` | Transaction timing/behavior shall be validated for compatibility with the Raspberry Pi Pico (RP2040) hardware SPI peripheral, since APS6404L PSRAM modules are commonly hosted on RP2040-based boards and interop with that ecosystem is a stated goal. **Open — no concrete compatibility criteria (timing margins, supported modes) are documented yet; needs a specific checklist item.** |

## Key Functionality

| ID | Requirement |
|---|---|
| `GRPR-SPIM-005` | The block shall receive read/write commands from the CPU over AHB-Lite and translate them into SPI transactions per the APS6404L command set. |
| `GRPR-SPIM-006` | The block shall support `SPI_READ`, `FAST_READ`, `SPI_WRITE`, and `FAST_WRITE` commands. |
| `GRPR-SPIM-007` | The block shall drive the target external SPI device and support both read and write transactions. |
| `GRPR-SPIM-008` | The block shall expose a transaction complete  flag to the CPU. |
| `GRPR-SPIM-009` | `CPOL` and `CPHA` shall be independently programmable (SPI mode 0 or mode 3). |

## Register Map

| Offset | Name | Access | Reset | Purpose |
| --- | --- | --- | --- | --- |
| 0x00 | CTRL | R/W | 0x0000_0000  |  Static config: mode, clock, CS |
| 0x04 | CMD | R/W | 0x0000_0000   | Per-transfer descriptor + START |
| 0x08 | STATUS | RO | 0x0000_000A | Live state and FIFO flags |
| 0x0C | INT | W1C | 0x0000_0000   | Latched done/error events |
| 0x10 | ADDR | R/W | 0x0000_0000  | Address phase payload |
| 0x14 | DATA | R/W | —            | TX FIFO push (write) / RX FIFO pop (read) |

Unlisted bits are reserved: write 0, read 0.

## CTRL — 0x00

Written once at init. Not touched per transfer.

| Bits | Field | Access | Description |
| --- | --- | --- | --- |
| 0 | CPHA | R/W | Clock phase    |
| 1 | CPOL | R/W | Clock polarity |
| 9:2 | CLKDIV | R/W | SCLK = fclk / (2 × (CLKDIV + 1)) |
| 14 | IE_COMPLETE | R/W | Interrupt enable for INT.TXN_COMPLETE |
| 15 | IE_ERR | R/W |  Interrupt enable for INT error bits |

Writing CTRL while STATUS.BUSY = 1 is ignored and sets INT.CFG_ERR.

## CMD — 0x04

Write with START = 1 to launch a transfer. Single-store kickoff.

| Bits | Field | Access | Description |
| --- | --- | --- | --- |
| 0 | START | R/W | Self-clearing. Always reads 0 |
| 8:1 | OPCODE | R/W | Raw 8-bit opcode, sent when CMD_EN = 1 |
| 9 | CMD_EN | R/W | Emit command phase |
| 10 | ADDR_EN | R/W | Emit address phase from ADDR |
| 12:11 | ADDR_BYTES | R/W | 0 = 1 byte, 1 = 2, 2 = 3, 3 = 4 |
| 13 | DATA_EN | R/W | Emit data phase |
| 14 | DIR | R/W | 0 = write (drain TX FIFO), 1 = read (fill RX FIFO) |
| 19:15 | DUMMY | R/W | 0–31 dummy SCLK cycles between address and data |
| 27:20 | DATA_LEN | R/W | Data phase length in bytes, minus 1. Range 1–256 |

Phase order: CMD → ADDR → DUMMY → DATA. Any phase may be omitted.
LEN and DIR are ignored when DATA_EN = 0.

## STATUS — 0x08 (read-only)

| Bits | Field | Description |
| --- | --- | --- |
| 0 | BUSY | Transfer in progress |
| 1 | TX_EMPTY | TX FIFO empty |
| 2 | TX_FULL | TX FIFO full |
| 3 | RX_EMPTY | RX FIFO empty |
| 4 | RX_FULL | RX FIFO full |
| 9:5 | TX_LEVEL | TX FIFO occupancy |
| 14:10 | RX_LEVEL | RX FIFO occupancy |

Reset value 0x0A reflects TX_EMPTY = 1 and RX_EMPTY = 1.

## IRQ_STATUS — 0x0C (write-1-to-clear)

| Bits | Field | Description |
| --- | --- | --- |
| 0 | DONE | Transfer completed |
| 1 | OVERRUN | RX byte arrived with RX FIFO full |
| 2 | UNDERRUN | TX byte needed with TX FIFO empty |
| 3 | CFG_ERR | START while BUSY, illegal descriptor, or CTRL write while BUSY |

Interrupt asserts when (DONE & CTRL.IE_DONE) or ((OVERRUN | UNDERRUN | CFG_ERR) & CTRL.IE_ERR).

## ADDR — 0x10

| Bits | Field | Access | Description |
| --- | --- | --- | --- |
| 31:0 | ADDR | R/W | Address payload. CMD.ADDR_BYTES selects how many low-order bytes are sent, MSB first within the field |

For a 24-bit device address, set ADDR_BYTES = 2 and place the address in bits 23:0.

## DATA — 0x14

| Access | Behaviour |
| --- | --- |
| Write | Pushes 1–4 bytes into the TX FIFO |
| Read | Pops from the RX FIFO |

Reading an empty RX FIFO returns the last popped value and sets INT.UNDERRUN.
Writing a full TX FIFO is dropped and sets INT.OVERRUN

## Clocking Strategy

Single clock domain. SCK is generated by dividing the system clock (see `GRPR-SPIM-010` for the unresolved frequency/ratio question).

## Reset Strategy

`rst_n` active-low synchronous reset.

## CDC Strategy

Fully synchronous design. `MISO` is sampled on the rising bus clock edge; the source material notes MISO needs a 2-stage synchroniser located inside the GPIO Mux (i.e. this block does not handle that synchronization itself — see [GPIO Mux § CDC Strategy](GPIO%20Mux.md#cdc-strategy)).
