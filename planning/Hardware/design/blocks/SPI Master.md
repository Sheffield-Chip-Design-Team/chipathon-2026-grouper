# AHB SPI Master

**Owner:** Thiri
**Status:** RTL design starting (per [Schematic Review](../../Schematic%20Review.md) §4, one of the three peripherals — SPI M, SPI S, QSPI — where RTL design has begun). No RTL committed yet under `hw/rtl/`.

**Related:** [Grouper SoC Specification](../Grouper%20SoC%20Specification.md) — memory map, boot sequence (SPI M is a candidate post-boot storage path) | [SPI Master Verification Plan](../../verification/blocks/SPI%20Master%20Verification%20Plan.md)

---

## Purpose

SPI master peripheral that lets the PicoRV32 CPU issue SPI transactions to an external SPI/QPI memory device (APS6404L-compatible command set), via an AHB-Lite slave interface. Fits the boot-sequence note that SPI (and QSPI) may be used for extended external storage once the initial UART-loaded program is running (see [Grouper SoC Specification § Boot Sequence](../Grouper%20SoC%20Specification.md#boot-sequence)).

## Protocols / Standards Conformity

| ID | Requirement |
|---|---|
| `GRPR-SPIM-001` | AHB-Lite subordinate interface on the CPU side. |
| `GRPR-SPIM-002` | SPI mode 0 (and mode 3, see `GRPR-SPIM-006`), MSB-first, on the external device side. |
| `GRPR-SPIM-003` | Command/transaction set shall be compatible with the APS6404L datasheet's SPI-mode command encoding. |
| `GRPR-SPIM-004` | Transaction timing/behavior shall be validated for compatibility with the Raspberry Pi Pico (RP2040) hardware SPI peripheral, since APS6404L PSRAM modules are commonly hosted on RP2040-based boards and interop with that ecosystem is a stated goal. **Open — no concrete compatibility criteria (timing margins, supported modes) are documented yet; needs a specific checklist item.** |

## Key Functionality

| ID | Requirement |
|---|---|
| `GRPR-SPIM-005` | The block shall receive read/write commands from the CPU over AHB-Lite and translate them into SPI transactions per the APS6404L command set. |
| `GRPR-SPIM-006` | The block shall support `SPI_READ`, `FAST_READ`, `SPI_WRITE`, and `FAST_WRITE` commands. |
| `GRPR-SPIM-007` | The block shall drive the target external SPI device and support both read and write transactions. |
| `GRPR-SPIM-008` | The block shall expose a busy flag to the CPU. |
| `GRPR-SPIM-009` | `CPOL` and `CPHA` shall be independently programmable (SPI mode 0 or mode 3). |

## Block Diagram

Micro-architecture based on one or two shift registers. Internal sub-blocks (per the Schematic Review's SPI M block diagram): control/status registers (`SR_TARGET`, `SR_ADDR`, `SR_DATA`, `SR_CTRL` — start & busy), an FSM (`IDLE`, `CS_SETUP`, `TRANSFER`, `DONE`), a clock divider, a bit counter, and a MISO latch (captures data on the rising SCK edge). External outputs: `MOSI`, `SCK`, `CS`, and input `MISO`.

**Note:** `SR_TARGET` was presumably a device-select register for addressing multiple SX1257s in the source material this was drafted from; with that context removed, its role here (if any — e.g. selecting among multiple external SPI devices) is an open item, not a confirmed requirement.

## Parameters and Configurations

| ID | Requirement |
|---|---|
| `GRPR-SPIM-010` | SCK shall be generated by a programmable clock divider from the system clock. **Open — the specific divide ratio and source-clock frequency are inconsistent across the source material: the checklist text says "4 MHz from 32 MHz (÷8)", the Schematic Review's own SPI M block diagram says "Clock divider (16 MHz → xMHz)", and the current bring-up RTL's default core clock (`picorv32_hello_core`) is 10 MHz. This must be resolved against the real top-level clock plan — see [Grouper SoC Specification § Clocking / Reset Architecture](../Grouper%20SoC%20Specification.md#clocking--reset-architecture) — before this block's clock divider is finalized.** |
| `GRPR-SPIM-011` | Shift-register width shall be configurable. |
| `GRPR-SPIM-012` | Clock divider ratio, `CPOL`, and `CPHA` shall all be programmable via registers. |

## IOs and External Interfaces

Grouped logically: AHB-Lite (CPU side), SPI external (`MOSI`/`MISO`/`SCK`/`CS`), and control/status registers. Additional IRQs may be added later (not yet specified). Exact external pin ownership depends on the still-undefined [GPIO Mux](GPIO%20Mux.md) pin-sharing scheme — see that block's open items.

## Clocking Strategy

Single clock domain. SCK is generated by dividing the system clock (see `GRPR-SPIM-010` for the unresolved frequency/ratio question).

## Reset Strategy

`rst_n` active-low synchronous reset.

## CDC Strategy

Fully synchronous design. `MISO` is sampled on the rising bus clock edge; the source material notes MISO needs a 2-stage synchroniser located inside the GPIO Mux (i.e. this block does not handle that synchronization itself — see [GPIO Mux § CDC Strategy](GPIO%20Mux.md#cdc-strategy)).

## Performance Targets

| ID | Requirement |
|---|---|
| `GRPR-SPIM-013` | Default SPI clock 4 MHz, 10 MHz max (subject to the `GRPR-SPIM-010` clock-plan resolution). |
| `GRPR-SPIM-014` | A 16-bit transaction shall complete within the time budget implied by the 4 MHz default clock. |

## Size Estimate

`GRPR-SPIM-015`: 1,500–2,000 gate equivalents (GE), per the source deck. Not yet confirmed by synthesis.

## Open Items

- `GRPR-SPIM-004` — Pico/RP2040 SPI compatibility has no concrete pass/fail criteria yet.
- `GRPR-SPIM-010` — three different clock-frequency figures in the source material (32 MHz text, 16 MHz diagram, 10 MHz current RTL default) need reconciling against the real top-level clock plan.
- `SR_TARGET` register's purpose needs re-scoping now that multi-SX1257 addressing doesn't apply (see Block Diagram note above).
- External pin ownership depends on the unresolved [GPIO Mux](GPIO%20Mux.md) pin-sharing scheme.
- IRQ signals "may be added later" — not yet specified.
